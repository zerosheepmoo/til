# `chmod`

## Overview

- Unix-like OS 의 쉘 명령어
- 파일이나 디렉토리의 접근 권한을 바꿈
- `change mod`의 약자

## Usage

- permission 보기

```zsh
ls -l filename
stat -c %a filename
```

- 사용

```zsh
chmod [options] mode[,mode] file1 [file2 ...]
```

### Option

- `-R`: 리커시브
- `-v`: 자세히

### Octal mode

> 예시: `drwxrwx---`
>
> numerical: `770`

- `d`: 디렉토리인지 여부
- `r`: 읽기 권한 `4`
- `w`: 쓰기 권한 `2`
- `x`: 실행 권한 `1`
- d 제외는 [class](https://en.wikipedia.org/wiki/File-system_permissions#Classes)
  - 왼쪽 세글자: user class
  - 가운데 세글자: group class
  - 오른쪽 세글자: other class
- numerical 에서 홀수는 실행, 4보다 크면 읽기, 홀수라면 1을 빼고 / 아님말구 상태로 `2`나 `6`이면 쓰기 보장

```zsh
chmod 777 filename
```

- 위에 만약 `setuid`, `setgid`, `sticky` bits가 지정안되어있으면 다음과 같은 것

```zsh
chmod 0777
```

### Symbolic Mode

```zsh
chmod [references][operator][modes] file ...
```
